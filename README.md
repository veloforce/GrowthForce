# GrowthForce

> English version: [README.en.md](./README.en.md)

GrowthForce 是一个本地优先的 AI 内容运营工作台，面向希望用 AI 提升内容生产、发布协作和运营复盘效率的创作者、运营团队与合作伙伴。

它不是单点写作工具，也不是只执行某个脚本的自动化工具。GrowthForce 把内容运营拆成可协作、可扩展、可复盘的 Agent 工作流：从热点与竞品研究，到选题策略、内容创作、平台适配、发布辅助、数据采集和长期经验沉淀。

## 核心能力

- **内容研究与选题策略**：围绕账号定位、热点、竞品和用户素材，辅助生成更贴近平台语境的内容方向。
- **多平台内容生产**：支持小红书、公众号等内容场景，帮助生成草稿、发布包、封面/配图思路和平台化排版内容。
- **运营闭环沉淀**：围绕 `账号定位 → 内容生产 → 数据采集 → 复盘沉淀` 建立持续迭代机制，让历史表现和用户偏好逐步变成可复用的运营 playbook。
- **人机协同发布**：对外发布和真实互动动作强调用户确认，适合需要稳定、可控、可审阅工作流的团队。
- **本地优先工作空间**：用户素材、配置、对话和运行数据优先保存在本机，适合个人创作者和小团队长期使用。
- **可扩展 Agent 架构**：通过 Skill、Tool 和 Connector 扩展能力，让新的平台、新的运营阶段或新的内部流程可以逐步接入。

## 适合谁使用

- 希望把 AI 融入日常内容运营的个人创作者。
- 需要持续生产小红书、公众号等内容的运营团队。
- 正在探索 AI Agent 工作流、内容自动化和运营复盘系统的合作伙伴。
- 需要一个本地桌面端工作台来管理素材、对话、任务和平台工作流的团队。

## GrowthForce 的不同点

### 从“生成内容”走向“运营闭环”

普通 AI 写作工具通常停留在单次输出。GrowthForce 更关注完整运营链路：研究、生成、发布辅助、采集、复盘和经验沉淀。每次内容表现都可以成为下一轮创作的输入。

### 从“聊天”走向“可执行工作台”

GrowthForce 以桌面应用承载 Agent、浏览器能力、平台连接器、本地数据和任务调度。用户可以在同一个工作台里完成内容准备、平台适配和后续复盘，而不是在多个工具之间反复切换。

### 从“固定流程”走向“可扩展能力”

项目把能力拆分为三层：

- **Agent**：理解用户目标并组织任务。
- **Skill**：定义具体阶段的 SOP、边界和协作规则。
- **Tool / Connector**：提供原子能力和平台连接能力。

这种结构让 GrowthForce 可以在保持核心体验稳定的同时，逐步接入更多内容平台、内部工具和运营方法论。

## 架构概览

GrowthForce 是一个跨平台桌面应用，整体采用本地优先的多进程架构：

```text
Desktop App
├─ Renderer：用户界面、对话工作台、任务与结果展示
├─ Main Process：窗口、会话、数据、调度和平台运行时管理
├─ Agent Process：AI Agent 执行、Skill 编排和工具调用
├─ Skills：内容研究、创作、发布、采集、复盘等阶段能力
├─ Tools：文件、内容分析、图片、浏览器、自动化等原子能力
└─ Connectors：连接小红书、公众号等平台工作流
```

设计目标是让长任务不阻塞桌面应用，让平台能力、内容能力和本地数据各自保持清晰边界。

## 使用边界

GrowthForce 旨在帮助用户更高效地完成内容运营工作，而不是替代用户对内容质量、合规性和平台规则的判断。

使用时请确保：

- 对外发布的内容经过人工审阅。
- 互动、评论、发布等动作遵守对应平台规则。
- 不使用 AI 编造数据、案例、政策、来源或个人经历。
- 对账号、素材和配置数据进行妥善管理。

## 当前状态

- 更完整的内容运营闭环。
- 更稳定的平台连接器。

- 更清晰的任务调度与复盘体验。
- 更适合个人创作者和团队协作的桌面工作台。

# 账号诊断设计

## 目标

新增平台无关阶段 Skill `account-diagnosis-ops`。用户选中小红书或公众号账号并要求诊断、
体检、分析现状或给出改进方向时，先读取账号定位和最近 10 篇已发布内容，输出评分卡、
证据、优势、问题和改进优先级。

诊断是只读分析，不修改 Profile，不创建或更新 Run，不写 `history.md` 或 `playbook.md`。

## 账号定位 Gate

诊断开始前使用选中账号的稳定 AccountRef 调用 `content_profile_get`。Profile 已有信息足以
完成本轮诊断时直接使用，不重复询问或确认。Profile 缺失或缺少本轮诊断必要字段时，遵循
选中账号 system reminder 中统一的缺失处理规则：

- 前台手动对话必须调用 `AskUserQuestion`，且只询问本轮诊断必要字段；
- 自动化任务不得调用 `AskUserQuestion`，跳过缺失信息获取并继续有限诊断，同时说明缺失
  字段及受影响的判断；
- 未选择内容账号时停止诊断并提示用户先选择账号，不要求补充 Profile。

诊断 Skill 保持只读，不直接更新 Profile；需要持久化用户补充的信息时交给
`account-profile-ops`。

## 诊断框架

诊断采用六个维度，总分 100：

| 维度 | 权重 | 主要证据 |
|---|---:|---|
| 定位与主页清晰度 | 15 | Profile、主页简介、内容实际主题 |
| 内容聚焦与一致性 | 20 | 最近 10 篇主题、目标受众、内容支柱 |
| 内容质量与包装 | 20 | 标题、开头、结构、可读性、标签或摘要 |
| 传播表现 | 20 | 平台实际可读的阅读或公开互动数据 |
| 互动与受众共鸣 | 20 | 收藏、评论、分享、留言及评论语义 |
| 运营稳定性 | 5 | 发布频率、样本数量、表现波动 |

每个维度必须同时给出得分、事实证据、解释和置信度。缺失数据记为 `N/A`，不按零分处理；
按可评分权重归一化展示总分，同时展示数据覆盖率。可评分权重低于 60% 时不生成总评级。

评级固定为：

- 85–100：健康
- 70–84：可优化
- 50–69：偏弱
- 0–49：需优先调整

不得引用无来源的行业平均值，不把相关性表述为因果，不以单篇内容概括整个账号。

## 小红书数据

使用当前连接器账号的稳定 account ID 调用：

- `content_profile_get`：读取账号定位；缺失处理遵循账号定位 Gate，缺失字段本身不按零分。
- `$AGENTSTUDIO_XHS_CLI user-profile --user-id <accountId>`：主页资料、关注数、粉丝数、
  获赞与收藏以及主页可读取笔记。
- `$AGENTSTUDIO_XHS_CLI get-feed-detail`：只抽取高表现、中位表现、低表现各一篇详情，
  最多加载 30 条评论，并遵守 `xhs-explore` 的详情访问风控节奏。
- `content_analyze_corpus`：分析最近 10 篇可用正文或详情的标题、结构和写作模式。

公开数据只能支持内容和公开互动诊断。曝光、观看、点击率、主页访客、涨粉和真实转粉效率
属于创作中心私有数据，v1 不读取，也不得以公开互动数据代替。

## 公众号数据

### 已有指标能力

保留 `wechat_metrics_fetch(days)`：

- 调用 `/datacube/getarticlesummary` 和 `/datacube/getarticletotal`。
- 只查询最近 1–7 个完整自然日，默认 3 天。
- 返回微信原始列表，不负责与最近 10 篇文章匹配。
- 只能作为近期传播表现的可选证据，不能覆盖发布时间早于查询窗口的文章。

因此该工具不能独立完成最近 10 篇公众号账号诊断，但无需新增另一套指标工具。

### 新增已发布文章读取

在 `wechat_ops` 新增只读原子工具：

```text
wechat_published_articles_fetch(
  offset?: integer = 0,
  count?: integer = 10,          // 1-20
  includeContent?: boolean = true
)
```

工具调用 `/cgi-bin/freepublish/batchget`，返回：

- `totalCount`、`itemCount`；
- 每条发布消息的 `articleId`、`updateTime`；
- 展平后的文章：标题、作者、摘要、正文 HTML、规范化 Markdown、文章 URL、原文 URL、
  封面、评论设置和删除状态。

Skill 默认取未删除的最近 10 篇文章。多图文消息按其中每篇文章分别进入样本。

### 失败和降级

所有 `wechat_ops` 接口都可能失败，包括但不限于：

- AppID、Secret 或 access token 无效；
- IP 白名单、接口调用额度或频率限制；
- 账号主体、认证状态或接口权限不足；
- 微信回收发布接口权限；
- 日期窗口没有数据、数据未完成统计或低阅读量未达到统计门槛；
- 网络错误或微信服务异常。

处理规则：

1. 已发布文章成功、指标失败：完成内容诊断；传播表现和依赖指标的项标记 `N/A`。
2. 已发布文章失败，但本地存在可读 Run/历史文稿：只输出“有限诊断”，明确不代表线上全貌。
3. 已发布文章失败且没有本地内容证据：停止并说明当前无法完成公众号账号诊断。
4. 不把接口失败或空数据解释为账号表现差，不把缺失指标按 0 分，不估算或串用其他文章数据。
5. 报告必须列出失败接口、原始错误摘要和受影响维度。

### 与发布后收集的边界

- `content-collect-ops` 在发布后按单个 Run 定时调用 `wechat_metrics_fetch`，把匹配的原始快照
  写入该 Run 的 `metrics.md`。
- `account-diagnosis-ops` 面向账号级即时分析，读取最近发布文章，并可临时调用同一指标工具；
  它不写任何 Run、History 或 Playbook。
- 两者复用 `wechat_metrics_fetch`，但不存在共享写入：只有收集 Skill 写 Run。诊断不调用
  `content_run_record_collection_result`，收集也不调用 `wechat_published_articles_fetch`。

## 输出

固定输出：

1. 账号、平台、样本范围和数据覆盖率。
2. 六维评分卡及可选总评级。
3. 做得好的事项和逐条证据。
4. 做得不好的事项、影响和置信度。
5. 最近 10 篇表现分布与代表内容。
6. 未来 30 天优先级最高的 3–5 个改进动作。
7. 无法获取、无法判断和接口失败的数据。

## 测试用例

1. Skill 能被发现，frontmatter 名称和触发描述正确，且明确只读边界。
2. 未选择账号时停止且不询问 Profile；Profile 缺失时，前台手动对话调用
   `AskUserQuestion`，自动化任务跳过询问并继续有限诊断、说明受影响的判断。
3. 小红书不足 10 篇、空主页、互动字段为空和详情不可访问时正确降级。
4. 小红书不输出基于曝光的互动率，不声称读取创作中心指标。
5. `wechat_published_articles_fetch` 参数边界、发布消息解析、多图文展平、HTML 转 Markdown、
   删除状态和 `includeContent=false` 正确。
6. 公众号已发布文章接口失败、指标接口失败、部分成功和空数据时符合降级规则。
7. 数据覆盖低于 60% 时不输出总评级。
8. 工具 TypeScript 校验、构建、Skill discovery 和现有 smoke tests 通过。

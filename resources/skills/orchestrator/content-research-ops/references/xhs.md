# 小红书站内研究 SOP

##  登录校验
- 小红书站内搜索、详情和主页使用 `$AGENTSTUDIO_XHS_CLI`。
- 执行任何小红书业务命令前，遵循 system reminder 中的 XHS 前置登录检查规则。

## 热点或竞品研究
- 小红书业务命令及参数以 `xhs-explore` 的“CLI 命令签名”为准；详情所需的 `id` 和
  `xsecToken` 必须来自同一条 `feeds[]`。
- 小红书热点或竞品研究按其中的双排序搜索、3–5 篇代表样本、对比维度和批量详情节奏执行。
- 没有参考账号时竞品账号调研标记为 `skipped`，继续热点和事实研究。

### 热点追踪

同一关键词至少执行两组搜索：

1. `--sort-by 最新 --publish-time 一周内`，观察近期新增内容、重复出现的话题和表达方式。
2. `--sort-by 最多点赞`，观察已有高互动内容的标题、封面、结构和评论需求。

输出时区分“近期出现频率”和“历史高互动”，不得把搜索排序直接表述为平台全量热度。

### 竞品分析

- 默认选择 3–5 篇有代表性的公开笔记；结果不足时使用实际可访问数量并说明样本限制。
- 样本应覆盖高互动、近期发布和差异化表达，避免只取同一作者或同一模板。
- 对每篇记录标题、封面特点、开头 hook、正文结构、标签、公开互动和评论需求。
- 使用 Markdown 表格横向比较，并总结共性、差异、可借鉴点和不可直接复制的部分。
- 批量读取详情时每组最多 3 篇，组间随机等待 10–20 秒。

这些数量和等待是当前自动化的保守执行策略，不是小红书官方规则。

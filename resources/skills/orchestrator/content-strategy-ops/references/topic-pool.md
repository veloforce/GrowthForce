# 选题池规则

## 输入

- profile 工具返回的账号定位、目标用户、内容领域、选题边界和参考账号。
- playbook 工具返回的有证据历史规律。
- history 和历史 Run 中最近 30 天的选题与表现。
- 本轮 research 文档的热点、竞品和事实研究。
- 用户当前工作目录中实际采用的素材。

## 基础评分

- 热度：30%
- 账号相关度：40%
- 切入价值：30%

命中 profile 选题边界或禁区的候选直接淘汰。playbook 只作为有证据的加权信号，
不能推翻 profile。

## 历史去重

- 最近 7 天出现相似核心关键词：强降权，并明确标记近期已覆盖。
- 7 至 30 天出现相似核心关键词：弱降权，并标记月内已有相关内容。
- 超过 30 天：不因时间直接降权，但可参考历史表现调整角度。

热点选题和常青选题混排。热点不足时用与账号定位匹配的教程、方法论、经验总结、
问题解答或测评类常青选题补充。

## 平台差异

- 公众号增加 SEO 友好度、长文展开空间和推荐框架，更关注观点完整性、结构递进、
  阅读和分享价值。
- 小红书增加收藏价值、互动潜力和视觉素材可执行性判断，更关注开头 hook、短段落、
  场景感和可操作性。

## Topics 输出

每个候选至少包含选题、拟定标题、目标用户、核心观点、评分、历史去重标记、平台
专属判断、内容结构、素材需求、推荐理由与风险。

记录最终选中选题、选择原因和建议排期，并把 topics 返回给用户或下游创作阶段。
本规则不写 Run 文档；如后续需要持久化 topics，由创建阶段在已有 runId 时负责。
排期仅是建议，不创建定时任务。
